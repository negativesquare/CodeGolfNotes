# Python Files Overview


## 利用頻度の高いファイル

実際の利用頻度が高い順に並べると次の通り。
どのファイルから読むべきかの目安として。

1. **modules.py**
   使用するモジュール名を毎回タイプするのは非効率なため、ここからコピペする運用にしている。

2. **power.py**
   入力サイズや値が大きい問題は毎回必ず存在するため、その都度参照している。

3. **example.py**
   特に、冒頭にある入力処理は定型句として頻繁にコピペする(短いので手書きすることも多い)。

4. **graph_book.py**
   Union-Find や Fenwick Tree をよくコピペする。
   DFS / BFS は問題ごとの差異が大きいため、1から書いた方が早いことが多い。

5. **graph_01.py**
   利用実績はあるが、使用頻度の低いアルゴリズムが多いため全体としての出番は少なめ。

---

各ファイルの役割は以下の通り。

---

## `example.py`
一般的な入出力、組み込み型、組み込み関数に関するメモ。
後半には上記だけを用いた短いアルゴリズム集を収録。

---

## `FFT.py`
高速フーリエ変換（FFT）の実装コード。
せっかく書いたので残しているが、実際には **`from acl_cpp.convolution import*`** を使う方が短く、かつ高速。

---

## `graph_01.py`
グラフ関連コードのうち、`graph_book.py` に収録しなかった内容をまとめたもの。

---

## `graph_book.py`
書籍 **「競技プログラミングの鉄則」** を参考にして書いたグラフ関連コード。
書籍の内容を基にしつつ、**大きく発展させた実装**も含む。

---

## `modules.py`
AtCoder で使用可能で、かつ使い勝手の良いモジュールについての説明とメモ。

---

## `not_graph.py`
グラフ以外のアルゴリズムのうち、主観的に「`example.py` に入れるほど短くはない」もの。

---

## `power.py`
大きな数をできるだけ短く表記するために、累乗表記（`a**b`）を使う際の補助コード。
`a` と `b` の桁数 (1,2), (2,1), (1,1) の3パターンに対し、短く書ける候補を探索するコードを用意。

### 1. `a`・`b` の一方が1桁、他方が2桁の場合
（つまり `a**b` が**5文字**になるケース）
- 指定した閾値を超える数値を1桁側の値ごとに探索し、**各値について最初に条件を満たした組だけを取得**。
- 取得した組を**累乗結果の昇順**に並び替えて出力。

### 2. `a`・`b` がどちらも1桁の場合
- `a**b` が**10000以上**になる組を、値の昇順で列挙。
- 実行結果(をコメント化したもの)をファイル内にそのまま記載。
- **9999以下**はそのまま書けば4文字以内なので、累乗表記は使わない方針。
